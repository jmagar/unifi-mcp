## Architecture & Structure

### Directory Layout
```
unifi_mcp/
├── main.py              # Server entry point
├── server.py            # FastMCP server config & initialization
├── client.py            # UniFi controller HTTP client
├── config.py            # Configuration & logging setup
├── formatters.py        # Data formatting utilities
├── models/              # Enums & parameter validation
│   ├── enums.py         # 31 UnifiAction types, service routing
│   └── params.py        # UnifiParams model
├── services/            # Domain service layer
│   ├── base.py          # BaseService (error, result, validation)
│   ├── device_service.py
│   ├── client_service.py
│   ├── network_service.py
│   ├── monitoring_service.py
│   └── unifi_service.py  # Main coordinator/router
├── tools/               # MCP tool registration
├── resources/           # MCP resource registration (6 modules)
```

### Design Pattern: Service Layer Architecture
- **Unified Tool**: Single consolidated `unifi` action-dispatcher (replaces 31+ separate tools)
- **Service Routing**: `UnifiService` coordinator routes actions via enums
- **Domain Services**: Device, Client, Network, Monitoring services extend `BaseService`
- **Separation of Concerns**: Tools -> Models -> Services -> Client -> API

### Data Flow
```
HTTP Request -> FastMCP Tool (unifi action + params)
  -> UnifiService.execute_action()
  -> Route by action category (enum-based)
  -> Domain Service (Device/Client/Network/Monitoring)
  -> UnifiControllerClient (httpx async HTTP)
  -> Format & Return ToolResult
```

### Entry Points
1. **main.py**: `main()` - CLI/standalone, `create_app()` - ASGI deployment
2. **server.py**: `UniFiMCPServer` - FastMCP lifecycle handler
3. **tools/**: `@mcp.tool()` decorated functions
4. **resources/**: `@mcp.resource()` decorated functions
